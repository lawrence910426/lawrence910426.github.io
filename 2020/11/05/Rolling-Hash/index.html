    費馬小定理

    ap=amod  pa^p=a \mod p ap=amodp

    這個式子很重要，一定要記住喔。
    記不起來怎麼辦？你就想成他會繞一圈回來吧。

    模逆元

    我們可以計算出 ap−2a^{p-2}ap−2，姑且稱呼這個數字為 a−1a^{-1}a−1。
    根據費馬小定理，我們得知 ap=aa^{p}=aap=a，也就是 ap−1=1a^{p-1}=1ap−1=1。
    此時，如果我們將 a−1a^{-1}a−1 乘上 aaa，那麼，a−1a1=ap−2a1=ap−1=1a^{-1}a^{1}=a^{p-2}a^{1}=a^{p-1}=1a−1a1=ap−2a1=ap−1=1，你就能把 aaa 的影響給取消掉了！

    後綴陣列 / 後綴數組 / Suffix Array

    Suffix Array 是一個相當有效的字串工具，該陣列代表「把字串所有後綴按照字典序排好」。
    詳細請見板橋高中資培講義 - Suffix Array。

    二分搜尋法

    二分搜尋法是一個高效的演算法，具有 O(logN)O(log N)O(logN) 的演算時間複雜度。
    詳細請見板橋高中資培講義 - 基礎演算法。

    Rolling Hash 判斷相同 (基礎概念)

    給定你兩個字串 A,BA,BA,B，問你這兩個字串是否一樣？

    很顯然的，我們都能很輕鬆的在 O(∣A∣+∣B∣)O(|A| + |B|)O(∣A∣+∣B∣) 內解答這個問題，但是這樣太 Low 了，我們必須要有毒瘤精神，必須要用毒瘤方法把題目解開。

    首先我們計算 H(A)H(A)H(A) 以及 H(B)H(B)H(B)，再判斷 H(A)H(A)H(A) 是否等於 H(B)H(B)H(B)。如果相同，我們就宣稱兩個字串相同；如果不同，我們就宣稱兩個字串不同。

    蛤，你說這樣有可能會錯喔，Rolling hash 的精神就是「唬爛」啊。
    世界上沒這麼衰的事情啦，多改幾次 x,px,px,p 參數，就會唬爛到過了啦 ><

    Rolling Hash 前綴和 (基礎概念)

    給你一個字串 TTT，問你 H(T[a,b])H(T[a,b])H(T[a,b]) 是多少？

    很顯然的，我們都能很輕鬆的在 O(∣T∣2)O(|T|^2)O(∣T∣2) 內解答這個問題，但是這樣太 Low 了，我們必須要有毒瘤精神，必須要用毒瘤方法把題目解開。

    摘要

    如果執行一個 O(∣T∣)O(|T|)O(∣T∣) 的預處理，就能夠在 O(1)O(1)O(1) 的時間內回答問題。

    實作

    • 如果你對數學跟程式碼過敏，你可以只看摘要
    • 如果你對數學過敏，你可以跳到後面程式碼的部分
    • 如果你對程式碼過敏，你可以只看數學的部分

    為了方便計算 T[a,b]T[a,b]T[a,b]，我們定義 S[N]=H(T[0,N])S[N]=H(T[0,N])S[N]=H(T[0,N])。
    我們有個很簡單的數學式

    S[0]=0S[0] = 0 S[0]=0

    S[N]=S[N−1]x+T[N−1]mod  p     ∀1≤N≤∣T∣S[N] = S[N-1]x+T[N-1] \mod p \ \ \ \ \ \forall 1 \leq N \leq |T| S[N]=S[N−1]x+T[N−1]modp     ∀1≤N≤∣T∣

    這個步驟，我們稱作預處理，只需要 O(∣T∣)O(|T|)O(∣T∣) 的時間就能做完了。
    當我們要計算 H(T[a,b])H(T[a,b])H(T[a,b]) 時，我們計算

    S[b]−S[a]=∑i=0b−1Tixi−∑i=0a−1Tixi=∑i=ab−1Tixi=xaH(T[a,b])S[b]-S[a]=\sum_{i=0}^{b-1} T_ix^i - \sum_{i=0}^{a-1} T_ix^i = \sum_{i=a}^{b-1} T_ix^i = x^a H(T[a,b]) S[b]−S[a]=i=0∑b−1​Ti​xi−i=0∑a−1​Ti​xi=i=a∑b−1​Ti​xi=xaH(T[a,b])

    這時候問題來了，我們要怎麼把 xax^axa 的影響給去除掉呢？答案就是模逆元！
    計算 xax^axa 的模逆元 x−ax^{-a}x−a，並且計算

    x−a(S[b]−S[a])=x−a(xaH(T[a,b]))=H(T[a,b])x^{-a}(S[b]-S[a])= x^{-a} (x^a H(T[a,b])) = H(T[a,b]) x−a(S[b]−S[a])=x−a(xaH(T[a,b]))=H(T[a,b])

    這個步驟，我們稱作查詢，只需要 O(log p)O(log\ p)O(log p) 的時間，就能得到 Rolling Hash 了！
    由於 ppp 通常不是我們討論的重點，將複雜度稍微省略後，我們可以很懶惰的說查詢只需要 O(1)O(1)O(1)！

    Rolling Hash 比大小 (基礎概念)

    給定兩個字串 AAA, BBB，判斷 AAA 比較大，還是 BBB 比較大？

    很顯然的，我們都能很輕鬆的在 O(∣A∣+∣B∣)O(|A| + |B|)O(∣A∣+∣B∣) 內解答這個問題，但是這樣太 Low 了，我們必須要有毒瘤精神，必須要用毒瘤方法方法把題目解開。

    摘要

    如果執行一個 O(∣A∣+∣B∣)O(|A| + |B|)O(∣A∣+∣B∣) 的預處理，就能夠在 O(logmin⁡(∣A∣,∣B∣))O(log\min(|A|, |B|))O(logmin(∣A∣,∣B∣)) 的時間內回答問題。

    實作

    • 如果你對程式碼過敏，你可以只看摘要

    先分別對 A,BA,BA,B 做前綴和預處理，需要 O(∣A∣+∣B∣)O(|A| + |B|)O(∣A∣+∣B∣) 的時間。這時候，我們可以在 O(1)O(1)O(1) 的時間內得到 A[la,ra]A[l_a,r_a]A[la​,ra​] 以及 B[lb,rb]B[l_b,r_b]B[lb​,rb​]。

    如果 A<BA<BA<B，我們知道一定會存在一個 iii，使得 A[0,i]=B[0,i]A[0,i]=B[0,i]A[0,i]=B[0,i]，而且 A[i]<B[i]A[i] < B[i]A[i]<B[i]。此時，我們對 iii 二分搜，搭配 Rolling Hash 比對，即可判斷誰比較大。

    Rolling Hash 避免 Hash 碰撞 (基礎概念)

    摘要

    俗話說的好，一個便當吃不夠，不會吃兩個嗎？
    一個 PPP 會出錯，不會用 NNN 個 PPP 嗎？一個 XXX 會出錯，不會用 NNN 個 XXX 嗎？

    如果這 NNN 組參數算出來的答案都一樣，我們就知道這兩串字串相等；如果有一個不一樣，我們就知道這兩串字串不相等。

    NNN 取的越大，你計算的時間就越久，但是相對安全；NNN 取的越小，你計算的時間就越快，但是相對不穩定。

    數學表達

    • 如果你對數學過敏，你可以只看摘要就好

    假設在程式中，我們要判斷 nnn 個字串，利用 相同生日問題 的近似解，搭配泰勒展開式，我們知道碰撞的機率 CCC 大概是

    C≈1−exp⁡(−n22p)≈n22pC \approx 1 - \exp(- \frac{n^2}{2p}) \approx \frac{n^2}{2p} C≈1−exp(−2pn2​)≈2pn2​

    如果我們取越多組參數，又假設所有參數的碰撞都是獨立事件，我們可以得到

    C≈(n22p)NC \approx {(\frac{n^2}{2p})}^N C≈(2pn2​)N

    Rolling Hash KMP / Z algorithm (進階應用)

    給定你兩個字串 T,PT,PT,P，問你 PPP 在 TTT 中出現過幾次？

    很顯然的，我們都能很輕鬆的在 O(∣T∣+∣P∣)O(|T| + |P|)O(∣T∣+∣P∣) 內用 KMP 或是 Z algorithm 解答這個問題，但是這樣太 Low 了，我們必須要有毒瘤精神，必須要用 Rolling Hash 把題目解開。

    摘要

    問題可以在 O(∣T∣+∣P∣)O(|T| + |P|)O(∣T∣+∣P∣) 內解決。

    實作

    藉由 Rolling Hash 前綴和，我們先對 TTT 做預處理，再來，我們枚舉 iii，並查詢 T[i,i+∣P∣]T[i,i+|P|]T[i,i+∣P∣] 的值，如果 H(P)H(P)H(P) 等同 H(T[i,i+∣P∣])H(T[i,i+|P|])H(T[i,i+∣P∣])，我們就得知這兩串字串相等，也就能夠把這題做完了！

    Rolling Hash Manacher (進階應用)

    給定一個字串 TTT，問你奇數長度，最長的回文有多長？

    很顯然的，我們都能很輕鬆的在 O(∣T∣)O(|T|)O(∣T∣) 內用 Manacher 解答這個問題，但是這樣太 Low 了，我們必須要有毒瘤精神，必須要用 Rolling Hash 把題目解開。

    摘要

    問題可以在 O(∣T∣ log ∣T∣)O(|T|\ log \ |T|)O(∣T∣ log ∣T∣) 內解決。

    實作

    假定 SSS 倒過來的字串叫做 S′S^\primeS′，也就是說 S′[i]=S[−i]S^\prime[i]=S[-i]S′[i]=S[−i]。

    枚舉 iii，對於 T[i]T[i]T[i]，我們可以藉由 Rolling Hash 判斷 T[i,i+K]T[i,i+K]T[i,i+K] 是否等於 T[i+1−K,i+1]′T[i+1-K,i+1]^\primeT[i+1−K,i+1]′，對 KKK 二分搜，即可找到最大的 KKK，滿足 T[i+1−K,i+K]T[i+1-K,i+K]T[i+1−K,i+K] 為奇數長度的回文字串。

    Rolling Hash 最長共同子字串 (進階應用)

    給定字串 AAA, BBB，KKK 滿足 A[a,a+K]=B[b,b+K]A[a,a + K]=B[b,b + K]A[a,a+K]=B[b,b+K]，問你 KKK 最大能到多少？

    很顯然的，我們都能很輕鬆的在 O(∣A∣2∣B∣2)O(|A|^2|B|^2)O(∣A∣2∣B∣2) 內解答這個問題，但是這樣太 Low 了，我們必須要有毒瘤精神，必須要用 Rolling Hash 把題目解開。

    摘要

    如果 N=min⁡(∣A∣,∣B∣)N=\min(|A|,|B|)N=min(∣A∣,∣B∣)，問題可以在 O(N log N)O(N\ log\ N)O(N log N) 內解決。

    實作

    我們可以考慮對 KKK 二分搜，將 H(A[0,0+K]),H(A[1,1+K]),...H(A[0,0+K]), H(A[1,1+K]), ...H(A[0,0+K]),H(A[1,1+K]),... 丟入集合 SSS，判斷 H(B[0,0+K]),H(B[1,1+K]),...H(B[0,0+K]), H(B[1,1+K]), ...H(B[0,0+K]),H(B[1,1+K]),... 是否存在於 SSS 內，即可回答問題。

    Rolling Hash 後綴數組 (進階應用)

    給定字串 TTT，請問 TTT 的後綴數組？

    很顯然的，我們都能很輕鬆的在 O(∣T∣)O(|T|)O(∣T∣) 內解答這個問題，但是這樣太 Low 了，我們必須要有毒瘤精神，必須要用 Rolling Hash 把題目解開。

    摘要

    問題可以在 O(T log2 T)O(T\ log^2 \ T)O(T log2 T) 內解決。

    實作

    我們知道，後綴數組的精隨在於「排序字串 TTT 的所有後綴」。

    如果我們有辦法在 O(log T)O(log\ T)O(log T) 的時間內，判斷 T[i,∣T∣]T[i,|T|]T[i,∣T∣] 比較小，還是 T[j,∣T∣]T[j,|T|]T[j,∣T∣] 比較小，我們就能在 O(T log2 T)O(T\ log^2 \ T)O(T log2 T) 的時間內，使用倍增法解決這個問題。
